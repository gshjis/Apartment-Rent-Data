# preprocesing (preprocessing) — препроцессор данных

Пакет предоставляет интерфейс препроцессора, предназначенный для построения единого train/test пайплайна. Основной объект — класс `Preprocessor`.

Главная логика строится вокруг метода `pipeline`, который последовательно применяет шаги преобразований в корректном порядке.

## Общая модель: train vs test

- **`mode="train"`**: для обучаемых шагов выполняются методы `fit_*`, и состояние (маппинги/модели) сохраняется внутри объекта препроцессора.
- **`mode="test"`**: обучающие шаги не выполняются; применяются ранее обученные состояния через `transform_*`.

## Интерфейс метода `pipeline`

**Подпись (смысл параметров):**

`pipeline(df, *, mode="train", drop_columns_args=None, merge_columns_args=None, rare_categories_args=None, anomaly_args=None, lda_type="probabilities", lda_args=None)`

### Принимает

- `df` — таблица данных (DataFrame).
- `mode` — режим выполнения: `"train"` или `"test"`.
- `drop_columns_args` — настройки шага удаления колонок.
- `merge_columns_args` — настройки шага слияния двух колонок.
- `rare_categories_args` — настройки шага замены редких категорий.
- `anomaly_args` — настройки шага поиска аномальных городов.
- `lda_type` — тип LDA-преобразования: `"probabilities"`, `"weighted_correlation"` или `"ohe"`.
- `lda_args` — параметры для LDA (например, имя текстовой колонки и число тем).

### Что делает (порядок)

1) При необходимости удаляет заданные колонки.

2) При необходимости объединяет две колонки в новую (разделитель `|`, пропуски приводятся к пустой строке). Опционально удаляет исходные колонки.

3) При необходимости заменяет редкие значения категориальных колонок на `other`:
   - в `train` — обучает список редких категорий по долям встречаемости;
   - в `test` — использует сохранённый список.

4) При необходимости добавляет признаки аномальности для пары `city+state`:
   - в `train` — оценивает кандидатов на аномалии с учётом статистических критериев и выбирает аномальные `city_state`;
   - в `test` — применяет заранее найденный набор аномалий.
   - результат: формируются флаги `is_anomaly` и тип `anomaly_type`.

5) При необходимости выполняет LDA по тексту (в зависимости от `lda_type`):
   - `probabilities`: добавляет вероятности распределения по темам (колонки `topic_*`).
   - `weighted_correlation`: вычисляет веса тем и добавляет агрегированный признак `topic_weighted_correlation`.
   - `ohe`: строит словарь top-N слов из обученных тем и добавляет бинарные индикаторы `lda_word_<word>`.

### Возвращает

- Новый DataFrame с добавленными/изменёнными признаками согласно выбранным шагам.

## Интерфейсы обучаемых шагов (что именно обучается)

### `fit_rare_categories(columns, threshold=...)`

Обучает маппинг: для каждой указанной колонки вычисляет долю встречаемости значений, относит редкие к списку и сохраняет список редких значений.

`transform_rare_categories(df)` затем заменяет эти значения на `other`.

### `fit_anomaly_cities(city_col, state_col, price_col=..., ...)`

Обучает критерии аномальности для комбинации `city+state` на основе ценового признака и статистического тестирования неоднородности.

`transform_anomaly_cities(df)` затем добавляет:

- `is_anomaly` — индикатор того, что комбинация признана аномальной;
- `anomaly_type` — категориальный тип аномалии (например, premium/budget).

## LDA: что означают варианты `lda_type`

- **`probabilities`**: признаки тем как вероятности на объект.
- **`weighted_correlation`**: из тем берутся веса (корреляции с ценой) и усредняются в один итоговый признак.
- **`ohe`**: вместо вероятностей строятся разреженные бинарные индикаторы наличия top-N слов из тематического словаря.

## Сохранение и загрузка состояния

Интерфейс объекта поддерживает сериализацию:

- `save(path)` — сохраняет обученное состояние препроцессора.
- `load(path)` — восстанавливает препроцессор с сохранёнными параметрами.

После загрузки можно запускать `pipeline(..., mode="test")` с теми же шагами.
